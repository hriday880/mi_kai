## Session Context
Following the codebase audit, I am now applying the fixes to bring the project to a production-ready state.

## Reasoning: Asset Optimization (Decision 12)
The 24MB of PNG textures in `public/box/` were converted to WebP using the macOS `sips` utility. This format provides excellent compression for 3D textures while maintaining visual quality, drastically reducing the load time for the intro animation.

## Reasoning: Static Imports for basePath (Decision 13)
The contact page was using a hardcoded string `"/logo-gold.svg"`. I switched this to a Next.js static import (`import logo from '@/public/logo-gold.svg'`). Next.js's bundler automatically handles appending the `basePath` (`/mi_kai`) to static imports, ensuring the image loads correctly when deployed to GitHub Pages.

## Reasoning: i18n and Accessibility
- Added missing translation keys for the contact form error state and footer links.
- Upgraded the footer's legal placeholders to actual Next.js `<Link>` components (pointing to `#` for now, but semantically correct).
- Added `role="button"` and keyboard support (`onKeyDown`) to the scroll-to-top logo in the footer to ensure it is accessible to keyboard users.

## Reasoning: TypeScript Safety
Created an interface `SceneData` in `IntroAnimation.tsx` to replace the loose `any` type on `sceneDataRef`. This ensures type safety for the Three.js objects (camera, lights, meshes) stored across renders and accessed during the GSAP animation timeline.
